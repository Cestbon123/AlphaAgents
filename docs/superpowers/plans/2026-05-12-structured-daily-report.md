# AlphaAgents 结构化日报计划

日期：2026-05-12

## 背景

MVP 要求结构化日报作为每日正式留痕。当前系统已经具备选股快照、持仓维护、每日复盘、实际操作记录和沉淀候选确认池，但还缺少一个将这些结果汇总成“当日盘后报告”的入口。

## 成功标准

1. 后端可以生成指定日期的结构化日报。
2. 日报包含：市场摘要、选股摘要、持股摘要、复盘摘要、沉淀摘要。
3. 日报写入 SQLite workflow 库，服务重启后可读取最新日报。
4. 前端可以触发生成日报，并展示最新日报。
5. 日报只做投研复盘留痕，不生成交易指令。

## 任务拆分

### 任务 1：后端日报生成与持久化

- 扩展 `DailyReport` 模型，增加 `report_text` 便于前端展示和复制。
- SQLite workflow 仓储新增 `daily_reports` 表。
- 新增保存、按日期读取、读取最新日报方法。
- 新增 `DailyReportWorkflow`，汇总本地行情状态、最新选股快照、持仓分析、实际操作记录和沉淀候选。
- 新增 `POST /api/v1/reports/daily/run`。
- 新增 `GET /api/v1/reports/daily/latest`。

### 任务 2：前端日报展示

- 新增“结构化日报”面板。
- 新增“生成日报”按钮。
- 页面初始化时读取最新日报。
- 生成后刷新展示最新日报。

### 任务 3：验证与文档

- 增加 API/仓储/前端静态测试。
- 运行完整测试和 JS 语法检查。
- 更新 `docs/project-status.md`。
