# Agent 对话历史管理 — 规格文档

## 背景

Agent 化改造完成后，每次聊天自动创建 session 并持久化消息到 SQLite。但前端缺少：
- 对话列表展示
- 历史对话查看/恢复
- 历史内容搜索

当前前端只用了一个 `agentSessionId` 变量，刷新页面就丢失历史。

## 目标

在 Agent Chat Panel 内实现完整的对话历史管理：查看列表、选择恢复、搜索历史、新建对话。

## 非目标

- 不做多用户隔离（单人使用）
- 不做对话导出/分享
- 不做对话删除（MVP 阶段先不做）
- 不改造后端的记忆存储结构（已有 `agent_sessions` + `agent_messages` + FTS5）

## 架构设计

完全复用现有基础设施，不改后端：

```
前端 Chat Panel
├── 对话头部栏 [当前对话名] [📋 历史] [➕ 新建]
├── 消息列表区
├── 输入区
└── 历史侧边栏（点击「📋 历史」弹出）
    ├── 搜索框
    ├── 对话列表（时间倒序）
    │   ├── 2026-06-02 "今天有什么关注的" · 4 条消息
    │   └── 2026-06-01 "分析 000001" · 8 条消息
    └── 点击任意对话 → 加载该 session 消息到聊天区
```

## 数据模型

复用已有表：

```
agent_sessions  — id, started_at, ended_at, summary
agent_messages  — id, session_id, role, content, timestamp
agent_messages_fts — content FTS5
```

## API 设计

复用已有端点，新增一个列表端点：

```
GET  /agent/sessions/{session_id}       → 已有，读取单次对话消息
POST /agent/chat  {session_id:"xxx"}   → 已有，续接指定会话
GET  /agent/sessions                     → 新增，返回对话列表
GET  /agent/sessions/search?q=xxx        → 新增，搜索历史消息
```

### GET /agent/sessions 响应

```json
{
  "sessions": [
    {
      "id": "xxx",
      "started_at": "2026-06-02T...",
      "summary": "用户: 今天有什么关注的 | agent: ...",
      "message_count": 6
    }
  ]
}
```

### GET /agent/sessions/search?q=xxx 响应

```json
{
  "results": [
    {
      "session_id": "xxx",
      "content": "...",
      "timestamp": "...",
      "role": "user"
    }
  ]
}
```

## 交互设计

### 对话头部栏

```
┌─────────────────────────────────────────────────┐
│ 💬 对话                          [📋] [+] [✕]  │
│ 2026-06-02 · 4 条消息                           │
├─────────────────────────────────────────────────┤
│ ...                                              │
```

- `[📋]` 历史按钮 → 侧边栏滑出
- `[+]` 新建对话 → 清空聊天区，创建新 session
- `[✕]` 关闭侧边栏

### 历史侧边栏

```
┌── 对话历史 ────────────────────┐
│ 🔍 [搜索历史内容...]            │
│                                │
│ 📅 2026-06-02                  │
│ ┌──────────────────────────┐   │
│ │ 今天有什么关注的？         │   │
│ │ 4 条消息 · 10分钟前      │   │
│ └──────────────────────────┘   │
│ 📅 2026-06-01                  │
│ ┌──────────────────────────┐   │
│ │ 分析 000001              │   │
│ │ 8 条消息 · 1天前         │   │
│ └──────────────────────────┘   │
└────────────────────────────────┘
```

- 点击对话卡片 → 加载消息到聊天区，关闭侧边栏
- 搜索框输入 → 实时过滤匹配的消息，显示上下文
- 搜索结果点击 → 加载该消息所在 session

### 消息持久化保持

- 前端每次发送消息前，如果已经有 `agentSessionId`，就带上续接
- 页面刷新后，默认显示最近一次对话（调用 `GET /agent/sessions` 取最新一条）
- 对话列表数据用 Promise 缓存，避免重复请求

## 技术决策与权衡

| 决策 | 选择 | 原因 |
|------|------|------|
| 列表端点 | 扩展现有 `/agent/` 路由 | 不改架构，最小改动 |
| 前端状态 | 全局变量 + DOM 渲染 | 项目没有前端框架，保持简单 |
| 搜索 | LIKE 中文 + FTS5 英文 | 兼容性，不引入额外分词器 |
| 实时刷新 | 无 | 单用户，手动刷新即可 |
