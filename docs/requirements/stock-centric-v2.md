# AlphaAgents 四入口个股驱动需求规格 v2

## 文档状态
- 状态：草案
- 阶段：四入口个股驱动重设计
- 范围：A 股投研、选股、复盘、沉淀和研究报告工作流
- 说明：本文定义产品需求和边界，后续前端、后端和测试均以本文为准。

## 产品目标
AlphaAgents v2 从“模块驱动”调整为“个股驱动”。用户每天进入系统后，先在“看盘选股”中找到需要关注的股票，再围绕当前股票完成分析、操作记录、复盘、沉淀和报告回看。

系统仅用于投研、复盘、分析和决策辅助，不提供交易执行、下单、撤单或券商委托能力。

## 信息架构
主导航只保留四个入口：
- 看盘选股：主工作页，合并原看盘和智能选股能力。
- 选股策略：配置结构化选股策略、查看策略参数，并通过 AI 生成可审查的策略草稿。
- 案例库：复盘案例和沉淀经验的回看与管理视图。
- 分析报告：TradingAgents/DeepSeek 生成的研究报告列表和详情视图。

复盘和沉淀不再作为主导航入口。它们是当前股票上的动作，汇总管理放入案例库。

## 个股驱动原则
系统必须维护统一的 `currentSymbol`。任何入口点击股票，都切换当前股票上下文：
- 看盘选股中的板块/行业/概念筛选结果、K 线标题和 Agent 洞察。
- 选股策略中的策略命中说明和执行结果。
- 案例库中的案例卡片、复盘来源和沉淀记录。
- 分析报告中的报告列表项和报告详情股票信息。

切换 `currentSymbol` 后，页面应读取或刷新同一个 `StockWorkspace`：
- 股票代码、名称、市场和最新 K 线摘要。
- 最新选股命中与策略原因。
- 持仓状态和持股分析结果。
- 最新 TradingAgents 风格研究报告。
- 操作记录、复盘案例、沉淀经验和跟踪状态。
- 数据缺口说明。

## 看盘选股
看盘选股是默认首页和主工作页。

页面需要包含：
- 左侧主导航：看盘选股、选股策略、案例库、分析报告。
- 顶部搜索：支持输入股票、代码和板块。
- 数据状态条：优先展示当前数据日期、当前策略和当前股票，市场指数不作为首页核心区域。
- 筛选区：只保留板块、行业、概念入口；板块元数据来自本地通达信缓存同步到 SQLite，暂不接入 TdxQuant。
- 股票入口区：按列表展示股票名称、现价、今日涨跌幅，点击行切换当前股票。
- 当前股票 K 线区：主图 K 线、知行趋势线和多副图指标。
- Agent 洞察区：展示基本面、技术面、资金面、风险审查等专家摘要。
- 当前股票动作区：生成分析、记录操作、写复盘、沉淀经验、更新跟踪状态。
- 当前股票时间线：选股命中、研究报告、操作记录、复盘案例、沉淀经验。

筛选结果列表必须能直接切换 `currentSymbol`。选股策略不使用通达信公式运行时，第一版采用结构化参数 + Python 执行器；AI 一键写策略只能生成可审查草稿，不执行任意代码。

## 选股策略
选股策略页用于配置当前选股逻辑，不作为交易入口。

选股策略页需要支持：
- 展示当前可用策略列表，第一版至少包含“知行趋势线”。
- 查看策略说明、执行引擎、启用状态、参数和规则。
- 编辑参数并持久化保存，使后续“执行选股”使用最新参数。
- 通过 DeepSeek/OpenAI 兼容接口生成策略草稿；API Key 未配置时返回本地安全模板草稿。
- 草稿默认不启用，用户必须审查并保存后才影响选股。

策略形态：
- 当前不是通达信公式运行时，也不执行用户输入的 Python 代码。
- 当前策略以结构化 JSON 参数描述，由后端 Python 执行器解释。
- 后续如需要支持通达信公式，应新增公式解析/沙箱/校验层，不能直接执行未审查文本。

## 案例库
案例库是复盘和沉淀后的管理视图，不是主要写复盘入口。

案例库需要支持：
- 按全部、成功、失败、规则修正、风险提醒、模式识别筛选。
- 按股票代码、股票名称、案例标题、模式关键词搜索。
- 每个案例必须绑定 `symbol`。
- 每个沉淀候选可选绑定 `review_case_id`。
- 点击案例后切换 `currentSymbol`，并展示对应股票上下文或联动走势。

案例卡片需要展示：
- 股票名称和代码。
- 案例类型、复盘结论、结果状态和日期。
- 来源复盘或来源报告。
- 关键原因、沉淀内容摘要和当前确认状态。
- 打开个股工作台、查看复盘、确认沉淀等操作。

## 分析报告
分析报告页只展示系统生成的研究报告。

报告来源限定为：
- TradingAgents 风格多专家流程生成的个股研究报告。
- DeepSeek 或兼容 OpenAI API 的 LLM 输出。

分析报告页不展示财报文件、新闻文件、券商研报文件或外部公开报告库。基本面、公告、新闻等信息只作为生成报告时的数据上下文或证据摘要，不作为独立文件管理对象。

报告列表需要展示：
- 报告标题、股票代码、股票名称、报告类型和生成时间。
- 最终跟踪结论：重点跟踪、观察、暂不跟踪、放弃。
- 参与专家数量、风险标记数量和数据缺口数量。

报告详情需要展示：
- 最终结论、核心理由、证据摘要、风险标记和数据缺口。
- 多专家分段摘要：技术分析、资金面、题材板块、消息公告、基本面、多方、空方、风险审查、研究经理。
- 报告正文 Markdown。
- 打开个股工作台、写复盘、沉淀为案例等非交易动作。

## 后端边界
后端以现有 `StockWorkspace` 为个股聚合核心。后续接口可补充轻量列表能力，但不得新增交易执行接口。

后端需要保证：
- 操作记录、复盘案例、沉淀候选和研究报告均绑定股票。
- 案例库可按股票、状态、类型读取复盘和沉淀数据。
- 报告列表只返回 AlphaAgents 生成的 TradingAgents/DeepSeek 个股研究报告。
- 数据缺失时接口仍返回可用空状态，并在 `data_gaps` 中说明。

## 非目标
- 不做自动交易、下单、撤单或券商委托。
- 不做外部公开研报文件管理。
- 不做财报文件管理中心。
- 不做新闻文件管理中心。
- 不引入 React 或大型前端框架作为 v2 的前置条件。
- 不新增独立股票详情页；个股工作台以内嵌核心区存在。
