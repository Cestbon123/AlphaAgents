## 改动详情

### ✅ 2026-05-13 通达信公式风格策略配置（重构）

**涉及文件：**
- `frontend/scripts/app.js` — `strategyFormulaBlocks` 数据结构增加 desc/hints 字段；`renderStrategyDetail()` 改为策略思想 + 条件行 + 公式预览三段式布局
- `frontend/styles/app.css` — 删除旧卡片样式，改为线条分割风格（无框、无背景、`border-bottom` 分割）
- `frontend/index.html` — 更新 CSS/JS 版本号强制缓存刷新

**说明：**
第一版改成卡片风格不符合项目「无卡片、线条分割」的设计规范，参照用户提供的 `invest_strategy_card_demo.html` 重新设计：

每个条件独立一行，从上到下依次展示：
1. **策略思想** — 策略核心逻辑说明 + 流程步骤（长期多头→短线回调→KDJ超卖→波动收敛→候选输出）
2. **条件行** — 标题 + 说明文字 + 公式表达式 + 可调参数输入 + 参考刻度（如"严格 10 / 标准 15 / 宽松 20"）
3. **固定条件** — 灰色展示趋势线对比和排除项
4. **公式预览** — 底部展示完整通达信公式代码块（深色背景 monospace 高亮）

### ✅ 2026-05-13 重建虚拟环境

**涉及文件：**
- `.venv/` — 用 `uv venv` 重建虚拟环境，`uv sync` 安装全部依赖

**说明：**
项目迁移到新路径后，旧的 `.venv/` 不存在。`start-dev.sh` 启动时检查 `.venv/bin/python` 不存在则报错退出。

修复方式：
- 使用 `uv venv` 基于当前 Python 3.12.3 重建虚拟环境
- 利用已有的 `uv.lock` 锁定文件执行 `uv sync`，精确还原依赖版本
- 无需修改 `start-dev.sh` 脚本本身（路径计算基于 `BASH_SOURCE` 动态解析，迁移后自动适配）

## 部署/运行说明

- 项目迁移后必须重新创建 `.venv` 虚拟环境并安装依赖，否则 `start-dev.sh` 会因找不到 Python 而退出。
- 重建命令：
  ```bash
  uv venv .venv
  uv sync
  ```
