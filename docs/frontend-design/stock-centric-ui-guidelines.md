# AlphaAgents 个股驱动前端设计指导

## 设计目标
AlphaAgents v2 前端采用“深色量化终端”方向：高信息密度、低装饰、强扫描效率。界面应让用户快速回答三个问题：
- 今天该关注哪只股票？
- 当前股票的证据、风险和数据缺口是什么？
- 这只股票下一步要记录、复盘、沉淀还是继续跟踪？

前端不得做营销页、介绍页或大型 Hero。打开应用后第一屏就是可操作的看盘选股工作区。

## 技术路线
- 继续使用原生 HTML/CSS/JS，不迁移 React。
- 继续使用本地 KLineCharts 渲染 K 线和指标。
- 不使用 CDN，不从外部加载组件、字体或图标。
- 以模块化 JS 函数和 CSS 组件类组织交互与样式。
- 保留后续迁移空间，但 v2 实现不得依赖 shadcn/ui、Tailwind 或 Vite。

## 图标方案
图标采用 Lucide 风格：线性、单色、24px 网格、2px 描边、圆角端点。

落地方式：
- 优先使用本地 SVG symbol sprite 或内联 SVG helper。
- 不直接从 CDN 引入 lucide。
- 每个图标按钮必须有可访问名称：`aria-label` 或可见文本。
- 主导航推荐图标：
  - 看盘选股：LayoutDashboard 或 SearchCheck 风格。
  - 选股策略：SlidersHorizontal 风格。
  - 案例库：CopyCheck 或 Library 风格。
  - 分析报告：FileText 风格。

## 组件规范
所有组件使用自研轻量 CSS/JS 实现，保持小圆角和可扫描信息密度。

基础组件：
- 侧边导航：固定窄栏，四个主入口，当前入口绿色高亮。
- 顶部栏：页面标题、搜索框、运行状态、主要动作按钮。
- 按钮：主按钮绿色，次按钮蓝灰，风险按钮红色；按钮高度固定，避免文字换行撑开布局。
- Tabs：用于报告类型、案例类型、K 线周期和筛选结果排序。
- 股票列表：用于看盘选股结果，只展示股票名称、现价、今日涨跌幅三列，点击行切换当前股票。
- 策略配置面板：显示启用状态、参数输入、规则说明和 AI 草稿生成入口。
- 专家卡片：显示角色、结论、证据摘要、置信度、风险标记。
- 案例卡片：显示股票、类型、结果、复盘日期、沉淀状态。
- 报告列表项：显示标题、类型、股票、生成时间、最终结论和数据缺口数量。
- 时间线：按时间展示选股命中、研究报告、操作记录、复盘案例和沉淀经验。
- K 线容器：必须给主图和副图稳定高度，副图分层展示，不能互相挤压。

## 色彩与视觉
色彩应围绕专业终端感，不使用一整屏单色渐变。

推荐设计变量：
- 背景：`#060b16`、`#0b1220`
- 面板：`#111827`、`#151f2e`
- 边界：`rgba(148, 163, 184, 0.18)`
- 正向/主操作：`#22c55e`
- 风险/下跌：`#ef4444`
- 警示/观察：`#f59e0b`
- 信息/选中：`#3b82f6`
- 正文：`#e5edf6`
- 次级文字：`#94a3b8`

设计规则：
- 卡片圆角不超过 8px。
- 页面不使用装饰性渐变球、漂浮大卡片或营销式插图。
- 表格、卡片和图表之间用细边界分隔，避免重阴影。
- 红绿只表达市场涨跌、风险和主动作，不用于无意义装饰。

## 四页面布局
### 看盘选股
看盘选股是默认首页，合并原 Dashboard 和 Screener。

布局：
- 左侧：主导航。
- 顶部：标题“看盘选股 / Market Screener”、搜索框、交易状态、执行选股按钮。
- 上方：数据状态条，优先显示数据日期、当前策略和当前股票。
- 左栏：板块/行业/概念筛选、板块搜索和当前策略摘要。
- 中间：当前股票 K 线、知行趋势线和副图指标。
- 右栏：Agent 洞察、跟踪状态和个股动作。
- 底部：股票列表和当前股票事件时间线。

交互：
- 点击任何股票卡片或列表项都调用统一 `setCurrentSymbol(symbol)`。
- “生成分析”调用当前股票研究接口。
- “写复盘”和“沉淀经验”必须绑定当前股票。

### 选股策略
选股策略用于配置结构化选股逻辑，不承载个股复盘或交易动作。

布局：
- 左侧：策略列表，显示策略名称、启用状态和执行引擎。
- 右侧：策略配置表单，包含启用开关、参数输入、规则说明。
- 下方：AI 一键写策略草稿输入区。

交互：
- 点击策略项加载对应配置。
- 保存策略调用后端策略配置接口，后续执行选股使用最新参数。
- AI 草稿默认只填充表单，不自动启用。
- 当前第一版策略形态是结构化 JSON 参数 + Python 执行器，不是通达信公式运行时。

### 案例库
案例库用于回看和管理复盘/沉淀，不作为主要写复盘入口。

布局：
- 顶部：标题“案例库 / Case Library”、搜索框、新建案例按钮。
- 筛选：全部、成功、失败、规则修正、风险提醒、模式识别。
- 主体：案例卡片网格。
- 下方：选中案例对应的联动 K 线与事件标记。

交互：
- 点击案例切换 `currentSymbol`。
- “打开个股工作台”回到看盘选股，并定位当前股票。
- “确认沉淀”只更新沉淀状态，不生成交易动作。

### 分析报告
分析报告只展示 AlphaAgents 生成的报告。

布局：
- 顶部：标题“分析报告 / Reports”、搜索框、新建报告按钮。
- 筛选：全部报告、技术面、资金面、题材、基本面、综合。
- 左侧：报告列表。
- 右侧：报告详情，显示最终结论、核心理由、证据、风险、数据缺口和多专家摘要。
- 报告详情只展示 TradingAgents/DeepSeek 个股研究报告，不承载外部文件管理。

交互：
- 点击报告切换 `currentSymbol`。
- 报告详情可打开个股工作台、写复盘、沉淀为案例。
- 不展示财报文件、新闻文件或外部公开报告文件入口。

## 响应式要求
- 桌面宽屏优先，保持三栏高密度布局。
- 中等屏幕允许右侧动作区下移，但当前股票 K 线必须仍然可见。
- 移动端以纵向堆叠为主：顶部搜索、当前股票摘要、K 线、动作区、列表。
- 任何文本不得溢出按钮、标签或卡片；长标题使用省略或换行策略。

## 可访问性与状态
- 所有可点击图标都必须有 `aria-label`。
- 运行中按钮必须有 disabled 状态和可见反馈。
- 数据缺口必须在界面可见，不允许用空白替代。
- 空状态必须说明下一步动作，例如“执行选股”“生成分析”“写复盘”。

## 禁止项
- 不使用 React、shadcn/ui、Tailwind 作为 v2 前置条件。
- 不使用 CDN。
- 不使用交易执行、下单、撤单、委托等动作或文案。
- 不把复盘和沉淀放回主导航。
- 不做财报文件、新闻文件、公开研报文件管理。
