# AGENTS.md

## 适用范围
- 本文件只定义代理行为约束与开发规范。
- 产品背景、需求、架构、流程和领域说明应放在 `README.md` 或 `docs/` 中维护。

## 核心原则
- 编码前先思考。不要默认假设；实现前先明确假设、歧义和权衡。
- 简洁优先。用最少的代码和最少的抽象解决问题，避免过度设计。
- 精准修改。只改本次需求必须改的内容，只清理因本次修改直接产生的无用代码。
- 目标驱动。先定义可验证的成功标准，再实现并验证，确认达成后再宣称完成。

## 开发规则
- 项目开发默认采用 Superpowers 插件的方法与技能流；涉及设计、计划、调试、TDD、验证、评审等环节时，优先使用对应的 Superpowers skill。
- 优先复用全局 `~/.codex/skills` 中已有的 skill，不要用随意拼接的 prompt 替代合适的 skill。
- 保持模块小而专，避免大文件、职责混杂和不必要的耦合。
- 设计与实现中应分离知识、策略、推理与提醒。
- 所有输出都必须结构化、可读、便于审查。
- 禁止硬编码路径、密钥和环境相关配置。

## 边界
- 本项目仅用于投研、复盘、分析和决策辅助，不得执行交易。
- 涉及风险、不确定性、假设和适用边界时，必须明确说明。

## 文档规范
- 开发文档一律使用中文，UTF-8。
- 持续变化的项目细节应放在 `README.md` 或 `docs/` 中维护。
- 本文件应保持简洁、稳定，只保留长期有效的工程规范。
